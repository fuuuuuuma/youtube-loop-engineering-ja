# youtube-loop-engineering-ja — CLAUDE.md

あなた（Claude Code）は、このキットを受け取った人の「**YouTube運用のうち、投稿以外を仕組み化したい**」を、
**ゴール型ループ（成功条件＋評価役で、合格するまで自動反復する）** で形にするエージェントです。
このファイルの手順を、他の一般指示より優先して守ってください。

---

## このキットの背骨（ゴール型ループ）

土台は Anthropic の [Getting Started with Loops](https://claude.com/blog/getting-started-with-loops)（2026-06-30）が説く、
4種類のループのうち **goal-based loop（②ゴール型）**——「成功条件を決めて、達成か上限ターンまで回す」——です。

> キモは「自分で採点しない」こと。**数値で測れる合格基準を先に決め、その基準を満たすまで自動で直させる。**
> 測れない部分（表現の妥当性・最終の耳確認・投稿の実行）は、必ず人間に残す。

くわしい考え方は [`reference/video-loop-blueprint.md`](reference/video-loop-blueprint.md)。ゴール型ループの雛形は
[`prompts/goal-loop-qc.md`](prompts/goal-loop-qc.md)。動くスキルは [`.claude/skills/youtube-loop/SKILL.md`](.claude/skills/youtube-loop/SKILL.md)。

---

## 最重要（絶対に守る）

1. **投稿は自動化しない。** このキットはリサーチ・台本・（任意で声/アバター）・サムネイル・QCまでを対象にする。
   YouTubeへのアップロード・公開ボタンは必ず人間が押す（BAN対策・意図的な設計）。
2. **合格基準は数値で決める。** 「なんとなく良い」ではなく、文字数・秒数・音量など、機械的に判定できる形にする。
3. **不合格の項目だけをやり直す。** 全部作り直さない。ピンポイント修正で1周を軽くする。
4. **重い依存を必須にしない。** 声クローン・実アバター・Whisper等は「あれば使う発展形」。無くても
   リサーチ→台本→サムネイル→QCの流れだけで動く前提で案内する。
5. **APIキーをコミットしない。** サムネイル生成（`gpt-image-2`）は `OPENAI_API_KEY` を環境変数で渡す。

---

## ワークフロー（3ステップ）

### ステップ1 — リサーチ＆台本
- ジャンル・URL・トピックを受け取り、Web検索等で一次情報を集める
- 章立て・ナレーション文・要点を含む構成データ（`script.json` 相当）にまとめる
- 数値・固有名詞は出典が取れたものだけ断定で書く。取れなければ「未確認」と明示する

### ステップ2 — 生成（動画・サムネイル）
- 声クローン・アバター口パクを使いたい場合は [`.claude/skills/youtube-loop/SKILL.md`](.claude/skills/youtube-loop/SKILL.md) の「発展形」を参照
- サムネイルは [`scripts/gen_thumbnail.py`](scripts/gen_thumbnail.py)（OpenAI API `gpt-image-2`）で複数案を生成する

### ステップ3 — QCループ（100点まで反復）
- [`prompts/goal-loop-qc.md`](prompts/goal-loop-qc.md) の雛形を、自分の動画の合格基準に書き換えて `/goal` に渡す
- 全項目が合格するまで自動で反復修正させる。数値化できない項目だけ、最後に人間が確認する
- QCが終わったら、**人間が最終確認してから手動で投稿する**（このキットはここで止まる）

# youtube-loop — YouTube運用ループ（投稿以外）簡略版

このスキルは、YouTube動画の**リサーチ・台本・サムネイル・QC**を、数値の合格基準で反復する
「ゴール型ループ」で進めるための簡略版です。声クローンや実アバターが無くても動きます。

## 前提

- 声クローン・mlx-whisper・Qwen3-TTS等の重い依存は**必須ではありません**（発展形として案内するだけ）
- サムネイル生成には `OPENAI_API_KEY`（OpenAI APIキー）が必要です
- **投稿（YouTubeへのアップロード・公開）はこのスキルの対象外**です。必ず人間が手動で行います

## ワークフロー

### 1. リサーチ

- ジャンル・トピック・参考URLを受け取り、Web検索で一次情報を集める
- 出典が取れた事実だけを断定で書く。取れなければ「未確認」と明示する
- 企画案（タイトル案・要点3〜5個）としてまとめる

### 2. 台本

- リサーチ結果を、章立て・ナレーション文の構成データに落とし込む
- 目安のフォーマット（`script.json` 相当）:

```json
{
  "title": "動画タイトル",
  "chapters": [
    {"heading": "導入", "narration": "..."},
    {"heading": "本編1", "narration": "..."}
  ]
}
```

### 3. 声・アバター（任意・発展形）

このスキル単体では声・アバターは生成しません。使いたい場合は次の考え方を流用してください。

- **声**: 音声合成を複数回実行し、ピッチのブレ（ジッター）など数値指標が最小のテイクを採用する「当たり采用」方式が安定します
- **テロップ改行**: ルールベースの形態素解析ではなく、LLMに文脈判断で改行させると自然になります
- **口パク**: 実写合成（Wav2Lipなど）は商用ライセンスに注意。イラストの口パターンを音量（RMS振幅）で切り替える方式なら¥0・商用利用OKで作れます

### 4. サムネイル作成

`scripts/gen_thumbnail.py` を使い、OpenAIの `gpt-image-2` で複数案を生成します。

```bash
export OPENAI_API_KEY=sk-...
python3 kit/scripts/gen_thumbnail.py --title "動画タイトル" --description "要点を1〜2文で" --out thumbnail.png
```

3案ほど生成し、目視で一番良いものを選びます。発展形として、「文字が読みやすいか」「被写体が中央か」等を
定量チェックにかけ、ゴール型ループで自動選定することもできます。

### 5. QC（ゴール型ループで100点まで）

`prompts/goal-loop-qc.md` の雛形を、自分の動画の合格基準に書き換えて渡します。基本の型は次の通りです。

1. **合格基準を数値で決める**（例: 1行25字以内、BGMは声に対し-19〜-25dB、無音区間は2秒以内）
2. **担当を分けて採点する**（音声担当・テロップ担当・効果音担当、のように役割を分けると1周が軽くなる）
3. **不合格の項目だけをピンポイントで直す**（全部作り直さない）
4. **全項目が合格するまで繰り返す**
5. **数値化できない項目（表現の妥当性等）だけ、最後に人間が確認する**

### 6. 投稿（このスキルの対象外）

QCが完了したら、**必ず人間がYouTube Studioで最終確認してから手動でアップロード・公開**します。
このスキルは投稿ボタンを押すところまでは対応しません（BAN対策として意図的な設計です）。

## やってはいけないこと

- 投稿・公開の自動化を実装しようとする
- 出典の無い数値・実績を断定で書く
- APIキーをコード・ファイルに直接書き込む
- QCをせずに「完成」と報告する

💡 「動画の企画を考えて」「サムネイル案を作って」等の依頼で自動的に発火します。

3サムネイル生成スクリプト（gen_thumbnail.py）

OpenAIのAPI（gpt-image-2）を叩いて、YouTubeサムネイル画像を生成する実際に動く最小Pythonスクリプトです。標準ライブラリのみで依存ゼロ。

  • OPENAI_API_KEY 環境変数が必要です（コードにキーは書き込まれていません）。
  • タイトルと説明を渡すだけでPNG画像を保存します。複数案の生成にも対応。
#!/usr/bin/env python3
"""GPT Image 2.0（gpt-image-2）でYouTubeサムネイル画像を1枚生成する最小スクリプト。

Claude Code から呼ぶ前提。OpenAI Images API を直接叩く（標準ライブラリのみ・依存なし）。

前提:
  export OPENAI_API_KEY=sk-...        # 必須（OpenAI Developers のキー）

使い方:
  python3 scripts/gen_thumbnail.py --title "YouTube自動運用ループ徹底解説" \
      --description "投稿以外を全部自動化する、という考え方の解説動画" \
      --out thumbnail.png

サイズ:
  YouTubeサムネイルの推奨比率は16:9。gpt-image-2 は 1536x1024（横長）が近い比率になる。

注意:
  - パラメータは OpenAI Images API に準拠。仕様変更時は body を調整する。
  - 画像内の日本語テキストは高精度だが、生成後に誤字・崩れを必ず目視で確認すること。
  - 複数案を生成して比較したい場合は --n を増やす（1枚ずつ連番で保存される）。
"""
import argparse
import base64
import json
import os
import sys
import urllib.error
import urllib.request

API_URL = "https://api.openai.com/v1/images/generations"
MODEL = "gpt-image-2"


def build_prompt(title: str, description: str) -> str:
    return (
        f"YouTubeサムネイル画像。タイトル「{title}」の内容が一目でわかる構図。"
        f"補足: {description}。"
        "文字は大きく読みやすいフォントで、背景とのコントラストを高くする。"
        "被写体は画面中央〜やや左寄りに配置し、右側にタイトル文字のスペースを空ける。"
        "誇張した表現・煽り立てる装飾は避け、清潔感のあるデザインにする。"
    )


def generate(prompt: str, size: str, out: str, quality: str, n: int) -> None:
    key = os.environ.get("OPENAI_API_KEY")
    if not key:
        sys.exit("OPENAI_API_KEY が未設定です。`export OPENAI_API_KEY=...` を実行してください。")

    payload = {"model": MODEL, "prompt": prompt, "size": size, "n": n}
    if quality and quality != "auto":
        payload["quality"] = quality
    req = urllib.request.Request(
        API_URL,
        data=json.dumps(payload).encode("utf-8"),
        headers={"Authorization": f"Bearer {key}", "Content-Type": "application/json"},
        method="POST",
    )
    try:
        with urllib.request.urlopen(req, timeout=300) as resp:
            data = json.load(resp)
    except urllib.error.HTTPError as e:
        sys.exit(f"API エラー {e.code}: {e.read().decode('utf-8', 'ignore')[:600]}")
    except urllib.error.URLError as e:
        sys.exit(f"接続エラー: {e}")

    items = data.get("data") or []
    if not items:
        sys.exit(f"画像データを取得できませんでした: {json.dumps(data)[:400]}")

    base, ext = os.path.splitext(out)
    ext = ext or ".png"
    os.makedirs(os.path.dirname(out) or ".", exist_ok=True)

    for i, item in enumerate(items):
        if item.get("b64_json"):
            img = base64.b64decode(item["b64_json"])
        elif item.get("url"):
            with urllib.request.urlopen(item["url"], timeout=300) as r:
                img = r.read()
        else:
            print(f"警告: {i}番目の画像データが取得できませんでした", file=sys.stderr)
            continue

        path = out if len(items) == 1 else f"{base}-{i+1}{ext}"
        with open(path, "wb") as f:
            f.write(img)
        print(path)


if __name__ == "__main__":
    ap = argparse.ArgumentParser(description="Generate YouTube thumbnail(s) with GPT Image 2.0")
    ap.add_argument("--title", required=True, help="動画タイトル")
    ap.add_argument("--description", default="", help="動画の要点（1〜2文）")
    ap.add_argument("--size", default="1536x1024", help="例 1536x1024（16:9に近い横長）")
    ap.add_argument("--out", required=True, help="保存先パス（例 thumbnail.png）。--n>1なら連番で保存")
    ap.add_argument("--quality", default="high", choices=["low", "medium", "high", "auto"])
    ap.add_argument("--n", type=int, default=1, help="生成する案の数（複数案を比較したいとき）")
    args = ap.parse_args()
    prompt = build_prompt(args.title, args.description)
    generate(prompt, args.size, args.out, args.quality, args.n)

💡 使い方: export OPENAI_API_KEY=... の後、python3 gen_thumbnail.py --title "..." --out thumbnail.png

4ゴール型ループの雛形プロンプト（goal-loop-qc.md）

動画のQC（品質チェック）を、Objective/Evidence/Validation/Done/Blockの5項目形式で「/goal」に渡すための雛形プロンプトです。合格基準を自分の動画に合わせて書き換えるだけで使えます。

  • 「なんとなく良い」ではなく、数値で判定できる合格基準を先に決めるのがコツです。
# Objective（目的）
<<動画ファイル名/パス>> の音声・テロップ・BGM/効果音を、下記の合格基準を全て満たす状態まで自動で直してください。

# Evidence（根拠・現状の実測方法）
- 音声: 無音区間の長さ、音量(RMS)のピーク・平均値をffmpegやPythonの音声解析ライブラリで実測する
- テロップ: 1行あたりの文字数、表示秒数、音声とのタイミングのズレを字幕ファイルから実測する
- BGM/効果音: 声のRMSに対する相対dB差を実測する

# Validation（検証方法・合格基準は数値で）
- 音声: 無音区間が <<2>> 秒を超える箇所が0件であること
- テロップ: 1行 <<25>> 字以内、表示秒数が読み上げ時間の80〜120%に収まっていること
- BGM/効果音: 声に対して <<-19〜-25>> dBの範囲に収まっていること
- 上記はすべて実測値で判定し、「なんとなく良い」という主観評価にしない

# Done（完了条件）
- 上記Validationの全項目が合格した時点で完了とする
- 合格した項目・実測値を一覧にして報告する
- 数値化できない項目（表現の不自然さ等）は、完了報告に「要目視確認」として明記し、自動で合格にしない

# Block（保留・確認が必要な条件）
- 合格基準を満たすために動画の構成・ナレーション内容自体を変える必要がある場合は、変更前に確認を取る
- <<上限ターン数（例: 6周）>> を超えても全項目が合格しない場合は、残課題を明示した上で一度停止する
- 投稿（YouTubeへのアップロード・公開）はこのループの対象外。QC完了後は必ず人間が最終確認してから手動で行う

💡 << >> の部分を自分の動画の合格基準に書き換えてから使ってください。

5汎用ブループリント（video-loop-blueprint.md）

特定のツールに依存しない、「リサーチ→台本→生成→QC→公開判断」のチェックリスト形式ブループリントです。Claude Code以外のエージェントでも流用できます。

  • チェックボックス形式なので、そのまま進行管理にも使えます。
■ ステップ1: リサーチ
□ トピック・ジャンルを1つに決める
□ 一次情報（公式サイト・SNS等）を最低2つ以上集める
□ 出典が取れた事実と、取れなかった推測を分けてメモする
□ 企画案（タイトル案・要点3〜5個）としてまとめる

■ ステップ2: 台本
□ 章立て（導入・本編・まとめ等）を決める
□ 各章のナレーション文を書く
□ 専門用語は、初めて出す箇所で日常語に言い換える
□ 元ネタにない主張を足していないか確認する

■ ステップ3: 生成
□ 声・映像・BGM等、必要な素材の生成方法を決める（自動生成 or 手動収録）
□ 各素材の「当たり/ハズレ」を判定する数値指標があれば決める
  （例: 音声のピッチのブレ幅、画像の解像度、字幕の文字数）
□ 複数案生成できるものは、1回勝負にせず複数回生成して比較する

■ ステップ4: QC（ゴール型ループ）
□ 合格基準を数値で決める（最初は1〜2個からで良い）
  例: 1行の文字数上限／BGMの相対音量／無音区間の長さ／サムネイルのコントラスト比
□ 担当を分けて採点する（音声担当・映像担当・テロップ担当、のように役割分担すると軽くなる）
□ 不合格の項目だけをピンポイントで直す（全部作り直さない）
□ 全項目が合格するまで「実行→検証→修正」を繰り返す
□ 数値化できない項目（表現の妥当性・全体の印象等）は、最後に人間が確認する

■ ステップ5: 公開判断（人間の仕事）
□ 完成品を人間が最終確認する
□ タイトル・説明文に誇張表現が無いか確認する
□ プラットフォームの利用規約・自動投稿ポリシーを確認する
□ 公開ボタンは人間が押す（自動化しない）

💡 「まず一番簡単な解から」——全部を一気に自動化しようとしないのがコツです。

受け取り方・使い方

  1. 各項目の「📋 コピー」ボタンでクリップボードにコピー、または「⬇ ダウンロード」でファイル保存
  2. GitHub上で kit/ フォルダを直接開き、コードブロックのコピーボタンを使うこともできます
  3. kit/CLAUDE.md をプロジェクト直下に配置し、Claude Codeでそのフォルダを開く

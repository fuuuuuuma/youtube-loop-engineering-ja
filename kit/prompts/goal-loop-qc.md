# 🎁 ゴール型ループ雛形プロンプト — 動画QCを100点まで回す

YouTube動画の品質チェック（QC）を、Anthropicの言う「ゴール型ループ」でそのまま回すための雛形です。

**受け取り方**: 下のコードブロック右上のコピーボタンでコピーするか、このファイルをそのままダウンロードしてください。

**使い方**: `<<>>` で囲んだ部分を自分の動画の合格基準に書き換えて、Claude Codeの `/goal` に渡してください。Objective（目的）／Evidence（根拠）／Validation（検証方法）／Done（完了条件）／Block（保留条件）の5項目形式です。

```text
# Objective（目的）
<<動画ファイル名/パス>> の音声・テロップ・BGM/効果音を、下記の合格基準を全て満たす状態まで自動で直してください。

# Evidence（根拠・現状の実測方法）
- 音声: 無音区間の長さ、音量(RMS)のピーク・平均値をffmpegやPythonの音声解析ライブラリで実測する
- テロップ: 1行あたりの文字数、表示秒数、音声とのタイミングのズレを字幕ファイルから実測する
- BGM/効果音: 声のRMSに対する相対dB差を実測する

# Validation（検証方法・合格基準は数値で）
- 音声: 無音区間が <<2>> 秒を超える箇所が0件であること
- テロップ: 1行 <<25>> 字以内、表示秒数が読み上げ時間の80〜120%に収まっていること
- BGM/効果音: 声に対して <<-19〜-25>> dBの範囲に収まっていること
- 上記はすべて実測値で判定し、「なんとなく良い」という主観評価にしない

# Done（完了条件）
- 上記Validationの全項目が合格した時点で完了とする
- 合格した項目・実測値を一覧にして報告する
- 数値化できない項目（表現の不自然さ等）は、完了報告に「要目視確認」として明記し、自動で合格にしない

# Block（保留・確認が必要な条件）
- 合格基準を満たすために動画の構成・ナレーション内容自体を変える必要がある場合は、変更前に確認を取る
- <<上限ターン数（例: 6周）>> を超えても全項目が合格しない場合は、残課題を明示した上で一度停止する
- 投稿（YouTubeへのアップロード・公開）はこのループの対象外。QC完了後は必ず人間が最終確認してから手動で行う
```

**ポイント**: 合格基準は最初から全部を厳密にしようとせず、まず1〜2個の数値基準から始めて、慣れたら増やすとうまくいきます。

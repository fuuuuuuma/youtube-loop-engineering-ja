# youtube-loop — YouTube運用ループ（投稿以外）簡略版

このスキルは、YouTube動画の**リサーチ・台本・サムネイル・QC**を、数値の合格基準で反復する
「ゴール型ループ」で進めるための簡略版です。声クローンや実アバターが無くても動きます。

## 前提

- 声クローン・mlx-whisper・Qwen3-TTS等の重い依存は**必須ではありません**（発展形として案内するだけ）
- サムネイル生成には `OPENAI_API_KEY`（OpenAI APIキー）が必要です
- **投稿（YouTubeへのアップロード・公開）はこのスキルの対象外**です。必ず人間が手動で行います

## ワークフロー

### 1. リサーチ

- ジャンル・トピック・参考URLを受け取り、Web検索で一次情報を集める
- 出典が取れた事実だけを断定で書く。取れなければ「未確認」と明示する
- 企画案（タイトル案・要点3〜5個）としてまとめる

### 2. 台本

- リサーチ結果を、章立て・ナレーション文の構成データに落とし込む
- 目安のフォーマット（`script.json` 相当）:

```json
{
  "title": "動画タイトル",
  "chapters": [
    {"heading": "導入", "narration": "..."},
    {"heading": "本編1", "narration": "..."}
  ]
}
```

### 3. 声・アバター（任意・発展形）

このスキル単体では声・アバターは生成しません。使いたい場合は次の考え方を流用してください。

- **声**: 音声合成を複数回実行し、ピッチのブレ（ジッター）など数値指標が最小のテイクを採用する「当たり采用」方式が安定します
- **テロップ改行**: ルールベースの形態素解析ではなく、LLMに文脈判断で改行させると自然になります
- **口パク**: 実写合成（Wav2Lipなど）は商用ライセンスに注意。イラストの口パターンを音量（RMS振幅）で切り替える方式なら¥0・商用利用OKで作れます

### 4. サムネイル作成

`scripts/gen_thumbnail.py` を使い、OpenAIの `gpt-image-2` で複数案を生成します。

```bash
export OPENAI_API_KEY=sk-...
python3 kit/scripts/gen_thumbnail.py --title "動画タイトル" --description "要点を1〜2文で" --out thumbnail.png
```

3案ほど生成し、目視で一番良いものを選びます。発展形として、「文字が読みやすいか」「被写体が中央か」等を
定量チェックにかけ、ゴール型ループで自動選定することもできます。

### 5. QC（ゴール型ループで100点まで）

`prompts/goal-loop-qc.md` の雛形を、自分の動画の合格基準に書き換えて渡します。基本の型は次の通りです。

1. **合格基準を数値で決める**（例: 1行25字以内、BGMは声に対し-19〜-25dB、無音区間は2秒以内）
2. **担当を分けて採点する**（音声担当・テロップ担当・効果音担当、のように役割を分けると1周が軽くなる）
3. **不合格の項目だけをピンポイントで直す**（全部作り直さない）
4. **全項目が合格するまで繰り返す**
5. **数値化できない項目（表現の妥当性等）だけ、最後に人間が確認する**

### 6. 投稿（このスキルの対象外）

QCが完了したら、**必ず人間がYouTube Studioで最終確認してから手動でアップロード・公開**します。
このスキルは投稿ボタンを押すところまでは対応しません（BAN対策として意図的な設計です）。

## やってはいけないこと

- 投稿・公開の自動化を実装しようとする
- 出典の無い数値・実績を断定で書く
- APIキーをコード・ファイルに直接書き込む
- QCをせずに「完成」と報告する
